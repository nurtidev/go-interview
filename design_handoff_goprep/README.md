# Handoff: GoPrep — дизайн-система и ключевые экраны

## Обзор
GoPrep — веб-платформа подготовки Senior Golang-разработчиков к собеседованиям (рынок СНГ). «Спортзал для собеседований»: методично, спокойно, без геймификации. Три механики, которые обязаны читаться в UI: учебник (главы 5–10 мин → «Прочитано» → «Закрепите материал»), вопросы с «лестницей глубины» (middle → senior → deep + самооценка SM-2), лайвкодинг по методике 25/5 (таймер, подсказки каждые 5 минут, разбор, возврат задач через 7/21/60 дней).

## О файлах в этом пакете
Файлы здесь — **дизайн-референсы, созданные в HTML** (прототип внешнего вида, не production-код). Задача — **воссоздать эти макеты в целевом стеке: React + Tailwind v4 + shadcn/ui**, используя токены ниже как CSS-переменные. HTML не копировать напрямую.

- `GoPrep Дизайн.dc.html` — единый канвас: токены, элементы, все экраны (открывается в браузере вместе с `support.js` и `ios-frame.jsx` из этой же папки).
- Внутри канваса каждый макет помечен id: `1a` токены, `1b` элементы, `1c/1d/1e` дашборд (2 десктоп-варианта + мобайл), `1f/1g/1h` страница вопроса, `1i/1j/1k` лайвкодинг, `1l/1m` учебник, `1n` повторение, `1o` статистика, `1p` вход. Секция `2a/2b/2c` — исторические цветовые пробы; **принят вариант 2c (графит), весь канвас уже переведён на него**.

## Fidelity
**High-fidelity.** Цвета, шрифты, размеры, отступы и копирайтинг — финальные, воспроизводить точно. Из пары вариантов одного экрана (напр. 1c/1d) реализуйте указанный в задаче; второй — референс альтернативной компоновки.

### Философия принятого направления (2c, «графит»)
- Бренд-цвета нет: акцент = чернила (`--ink`). Главные кнопки — чернильная заливка, текст `#FAF9F7`.
- Цвет несёт только смысл: зелёный = PASS/«Хорошо», красный = FAIL/«Снова», янтарь = «к повторению»/«Трудно»/фаза подсказок, синий = «Легко»/info, плюм = staff.
- Логотип: `GoPrep_` — «GoPrep» ink 600 + underscore `--ink-3` (курсор терминала).
- Иерархия — hairline-линиями (1px `--hairline`), не тенями и не карточками. Списки-строки вместо карточных сеток.
- Фон — тёплая «бумага» `#FAF9F7`, поверхности — белые. Никаких градиентов, неона, маскотов, эмодзи.
- Тёмная тема цельная (`.dark`); редактор кода — всегда тёмный остров `--editor` даже на светлой странице.

### Типографика
- UI: IBM Plex Sans — h1 24/600, h2 17/600, body 14/1.45, small 12.5, микрометки 11 (JetBrains Mono, uppercase, letter-spacing .08em, цвет `--ink-3`).
- Длинный контент (учебник, условия задач, вопросы, ответы): PT Serif 20/1.7 (заголовки статей 30/1.3 700; на мобайле 17.5/1.7 и 23/1.35). Читабельность — главная ценность, не ужимать.
- Код и все цифры-счётчики: JetBrains Mono (таймер — с `font-variant-numeric:tabular-nums`).
- Подключение: Google Fonts, subset cyrillic, weights: Plex Sans 400/500/600, PT Serif 400/700+italic, JB Mono 400/500/600.
- Спейсинг: база 4px → 4/8/12/16/24/32/48. Навбар 56px.

## Ключевые компоненты (спека — точные образцы в `1b`)
- **Бейджи сложности** (pill, 11.5/500, padding 3px 10px): middle — текст `--ink-2` на `#F0EDE7`; senior — `--ink` на `--accent-soft`; staff — `--staff` на `--staff-soft`.
- **Бейджи статусов**: «не начато» — контур 1px `--hairline`, текст `--ink-3`; «в процессе» — info-тинт; «✓ изучено» — ok-тинт; «к повторению» — warn-тинт.
- **Прогресс-бар**: высота 4px, трек `--hairline`, заливка `--ink`, radius 2px, без скруглённых концов заливки. Сегментный вариант (для глав): flex с gap 2px.
- **Таймер 25/5** (pill): фокус-фаза — белый фон, контур `--hairline`, зелёная точка 8px, время mono 18/600; после 25:00 — фон `--warn-soft`, янтарная точка, время `+MM:SS` цветом `#7A5A17`, рядом «подсказка через M:SS».
- **Кнопки самооценки** (4, pill, порядок фиксированный): Снова (err), Трудно (warn), Хорошо (ok), Легко (info) — текст семантическим цветом на его же -soft тинте; в кнопке вторая строка mono 10.5 с интервалом («10 мин / 1 день / 3 дня / 7 дней»). Задизейблены (opacity .55) до раскрытия всех уровней ответа.
- **Консоль тестов**: фон `--editor`, mono 12.5/1.75; PASS `#7DBE8E`, FAIL `#E08A80`, детали ошибки `#B9BEC4` с отступом, команда/итог `#8B9096`.
- **Таблица SQL**: рамка+разделители `--hairline`, radius 10, шапка mono 11/600 `--ink-3` на `#F5F3EE`, ячейки mono 12.5, футер «N строк · N мс».
- **SVG-схемы**: обводка 1.5px, заливки только -soft тинты, подписи JB Mono 10.5–11, стрелки `--ink-2`; акцентная заливка `--accent-soft` — только на главном узле. Подпись «Рис. N» 12px `--ink-3` по центру.
- **Empty state**: рамка 1px dashed `#D8D3C8`, radius 10, по центру глиф `[ ]` mono `#C9C3B7`, заголовок 14/500, пояснение 12.5 `--ink-3`, контурная pill-кнопка со следующим действием.
- **Кнопки**: primary — заливка `--accent`, текст `#FAF9F7`, pill, 14/500, padding 11px 22px; secondary — контур 1px `--accent`, текст `--accent`, фон none; на тёмном острове — светлая заливка `--editor-ink`/текст `--editor`.

## Экраны
Все размеры/отступы смотреть в HTML — ниже структура и поведение.

1. **Дашборд** (`1c` одноколоночный, `1d` с колонкой «Сегодня» 300px на `#F5F3EE`, `1e` мобайл). Приветствие + дедлайн собеседования, primary CTA «Повторить N вопросов», 3 счётчика через вертикальные hairline, список 5 секций вопросов (строка = название+подзаголовок, warn-бейдж «N к повторению», прогресс 160px + счётчик mono), блоки «Учебник» (текущая глава, сегментный прогресс) и «Лайвкодинг» (перерешать N + новая задача). Мобайл: CTA во всю ширину, 2 стат-карточки, секции списком, таб-бар 4 пункта.
2. **Учебник** (`1l` деск: оглавление 380px слева + колонка чтения 600px; `1m` мобайл-чтение). Оглавление: 2 темы, главы со временем чтения; прочитанные — ✓ зелёный + зачёркнутое серое; активная — строка с тинтом `--accent-soft`. Чтение: сериф, инлайн-SVG схема с подписью, в конце primary «✓ Прочитано», затем блок «Закрепите материал» (3 вопроса с бейджами сложности + кнопка). Мобайл: тонкий 3px прогресс чтения под шапкой, sticky-кнопка снизу.
3. **Страница вопроса** (`1f` — последовательное раскрытие: middle раскрыт, senior раскрыт, deep за кнопкой «Раскрыть уровень deep», самооценка задизейблена; `1g` — всё раскрыто, рельса-стёпер 1-2-3 слева, самооценка активна, выбранная кнопка получает border 2px своим цветом; `1h` мобайл — нераскрытый уровень виден заглушкой с fade, кнопка раскрытия закреплена снизу). Ответы — сериф 17/1.65, инлайн-код — mono 13 на `#F0EDE7`. Deep-уровень ссылается на главу учебника.
4. **Режим повторения** (`1n`): минимум хрома — «✕ Завершить», прогресс «3 / 7» по центру, оценка времени; вопрос сериф 25/1.4 по центру, кнопка «Показать ответ» → раскрывается вся лестница + самооценка.
5. **Лайвкодинг** (`1i` — условие 400px | редактор | консоль снизу, таймер в шапке, подсказки внизу условия: открытая — цитата с левым бордером 2px, будущие — с точками-статусами и таймером «через M:SS», «Открыть разбор» — после всех подсказок; `1j` — фаза сверх 25:00: условие свёрнуто в полосу на `#F5F3EE`, справа колонка «Лента 25/5» — вертикальный таймлайн событий (фокус → подсказка 1 → подсказка 2 через M:SS → разбор), внизу заметка о возврате через 7/21/60 дней; `1k` мобайл — вкладки Условие/Код/Тесты, счётчик упавших тестов в табе, таймер в шапке, нижняя панель с подсказкой и кнопкой «▸ Тесты»). Редактор: табы файлов (активный — underline 2px `--editor-ink`), номера строк `#5C6167`, подсветка: ключевые слова `#8FA5C8`, имена функций `#D8A44A`, комментарии `#7A8087`. Широкий код скроллится внутри блока (overflow-x:auto), страница горизонтально не скроллится никогда.
6. **Статистика** (`1o`): 4 числа через hairline (счётчики mono 30/600, серия — можно ink), heatmap 12 недель (7×12 квадратов 12px, ramp: `#F0EDE7 → #C4BEB2 → #7C766B → #26241F`), прогресс по секциям списком.
7. **Вход** (`1p`): центр 340px, логотип + слоган «Спортзал для собеседований Senior Go», email/пароль (input: фон white, border 1px `#D8D3C8`, radius 10, padding 11px 14px), primary «Войти», разделитель «или», secondary «Продолжить с GitHub», ссылка на регистрацию.

## Интеракции и состояния
- **Лестница ответов**: уровни раскрываются строго по одному сверху вниз; самооценка активируется после раскрытия всех; выбор оценки = запись SM-2 и переход к следующему вопросу (в режиме повторения).
- **Таймер 25/5**: 25:00 (45:00 для сложных) вниз → по нулю переключение в янтарную фазу, счёт вверх `+MM:SS`; каждые 5 минут разблокируется очередная подсказка; после всех — «Открыть разбор»; после разбора задача планируется на 7/21/60 дней.
- **Hover**: строки списков — фон `#F5F3EE`; pill-кнопки — насыщение тинта / `--accent-hover`; переходы 120–150ms ease-out. Ссылки — `--accent`, hover `--accent-hover`.
- **«Прочитано»** отмечает главу (✓ в оглавлении), открывает «Закрепите материал».
- Мобайл ≈390px: навигация — нижний таб-бар (Обзор/Учебник/Вопросы/Кодинг), хиты ≥44px, главные CTA закреплены снизу.
- Никакого горизонтального скролла страницы; код/таблицы скроллятся внутри своих блоков.

## Состояние (минимум)
- Пользователь: имя, дата собеседования, streak (текущий/рекорд).
- Учебник: массив глав {тема, заголовок, минуты, прочитана}.
- Вопросы: {секция, сложность, уровни ответов [3], SM-2 данные (due, interval, ease), раскрытые уровни}.
- Лайвкодинг: {язык Go/SQL, лимит 25/45, фаза таймера, открытые подсказки, результаты тестов, расписание перерешивания 7/21/60}.
- Тема: light/dark через класс `.dark` на html.

## Ассеты
Внешних ассетов нет: только Google Fonts (IBM Plex Sans, PT Serif, JetBrains Mono) и инлайн-SVG схемы, рисуемые по палитре диаграмм. Иконки в макете — простые геометрические заглушки; в реализации взять lucide-react (идёт с shadcn), обводка 1.5px.
