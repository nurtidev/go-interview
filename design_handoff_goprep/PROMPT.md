# Промпт для кодинг-агента

Скопируй текст ниже в своего агента (Claude Code и т.п.), положив папку `design_handoff_goprep/` в корень репозитория.

---

Реализуй UI платформы GoPrep по дизайн-пакету в `design_handoff_goprep/`.

Сначала прочитай `design_handoff_goprep/README.md` целиком — там дизайн-токены, спека компонентов, все экраны, интеракции и модель состояния. Файл `GoPrep Дизайн.dc.html` — hi-fi HTML-референс всех макетов (можно открыть в браузере из той же папки); это референс, а не код для копирования.

Стек: React + Tailwind v4 + shadcn/ui + lucide-react.

Требования:
1. Заведи токены из раздела «Дизайн-токены» README как CSS-переменные в глобальном CSS (`:root` + `.dark`) и подключи их в Tailwind v4 через `@theme`. Нигде не хардкодь цвета — только переменные.
2. Подключи шрифты (Google Fonts, cyrillic subset): IBM Plex Sans 400/500/600, PT Serif 400/700/400i, JetBrains Mono 400/500/600.
3. Собери базовые компоненты из раздела «Ключевые компоненты» (бейджи сложности/статусов, прогресс-бары, таймер 25/5, 4 кнопки самооценки, консоль тестов, SQL-таблица, empty state, кнопки) как переиспользуемые React-компоненты поверх примитивов shadcn — со всеми состояниями, описанными в README.
4. Реализуй экраны в порядке: дашборд (десктоп-вариант 1c + мобайл 1e), страница вопроса (вариант 1g с последовательным раскрытием как в 1f), лайвкодинг (1i, мобайл-вкладки как в 1k), затем учебник (1l/1m), режим повторения (1n), статистика (1o), вход (1p). Сверяй каждый экран с соответствующим блоком HTML-референса — совпадение должно быть пиксельным по типографике, отступам и цветам.
5. Соблюдай интеракции из README: лестница ответов раскрывается по одному уровню, самооценка активна только после полного раскрытия; таймер 25:00 → янтарная фаза `+MM:SS` с подсказкой каждые 5 минут; hover-состояния и переходы 120–150ms.
6. Мобайл — mobile-first брейкпоинт ~390px: нижний таб-бар, закреплённые CTA, хит-таргеты ≥44px. Горизонтального скролла страницы быть не должно; широкий код и таблицы скроллятся внутри своих блоков.
7. Тёмная тема — классом `.dark` (токены уже даны). Редактор кода — всегда тёмный остров `--editor`, в обеих темах.
8. Интерфейс на русском, копирайтинг бери дословно из макетов. Данные пока замокай в отдельном модуле по разделу «Состояние» README.

9. Реализуй лендинг (роут `/`) по спеке `design_handoff_goprep/LANDING.md` и референсу `GoPrep Лендинг.dc.html`: те же токены; liquid-glass эффекты строго в 4 местах из спеки (навбар-пилюля, hero-карточка, пилюля таймера, карточки отзывов) с fallback через `@supports not (backdrop-filter: blur(1px))`.

Не добавляй: градиенты, тени крупнее `--shadow-md`, эмодзи, карточные сетки там, где в макете списки с hairline-разделителями, и любые акцентные цвета кроме описанных семантических.
