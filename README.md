# GoPrep — подготовка к собеседованиям Senior Golang

Веб-продукт для системной подготовки к собеседованиям на Senior Golang разработчика (рынок СНГ: Яндекс, Ozon, Авито, Т-Банк и др.). Банк вопросов с «лестницей глубины», интервальное повторение по SM-2 и трекер прогресса.

## Идея: «лестница глубины»

У каждого вопроса три уровня ответа — как реально отвечают кандидаты разных грейдов:

- **middle** — корректный базовый ответ. Его хватит мидлу, но сеньора завалят фоллоу-апами.
- **senior** — механика изнутри, крайние случаи, грабли в проде. То, что отличает сеньора.
- **deep** — устройство в runtime, эволюция между версиями Go, связь со смежными темами.

Вы читаете вопрос, отвечаете вслух (как на интервью), затем раскрываете уровни один за другим и честно оцениваете себя: «Снова / Трудно / Хорошо / Легко». Алгоритм SM-2 назначает дату повторения — забытое возвращается чаще, усвоенное — реже.

## Запуск

Требуется Go 1.24+ и Node 20+ (только для сборки фронтенда).

```bash
# 1. Собрать фронтенд
make build-web

# 2. Запустить сервер (API + статика на одном порту)
make run
# → http://localhost:8080
```

Для разработки фронтенда — второй терминал: `cd web && npm run dev` (Vite-прокси перенаправит `/api` на :8080).

### Docker

```bash
docker build -t goprep .
docker run -p 8080:8080 -v goprep-data:/data goprep
```

| Переменная | Default | Назначение |

## Стек и решения

- **Бэкенд**: Go, stdlib `net/http` (роутинг 1.22+), SQLite (`modernc.org/sqlite`, без cgo), JWT, bcrypt. Минимум зависимостей — осознанно: это продукт для Go-разработчиков.
- **SM-2**: чистая реализация в `internal/srs` с unit-тестами.
- **Фронтенд**: Vite + React + TypeScript + Tailwind + shadcn/ui, тёмная тема.
- **SQLite вместо PostgreSQL** — выбор MVP ради нулевой настройки; интерфейс хранилища позволяет мигрировать позже.
